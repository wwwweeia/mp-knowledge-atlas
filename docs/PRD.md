# PRD: 微信公众号技术文章知识管理系统

> 状态: 方案设计中  
> 更新: 2026-05-21

---

## 背景

阿威长期通过微信阅读公众号技术文章，高质量文章会转发到个人聊天助手收藏。文章主题集中在：

- AI Coding / Claude Code
- Agent 架构 / Skills
- Harness Engineering / SDD 编程落地
- 知识管理 / RAG
- 行业思考

**现状**：已有 53 篇文章通过手动维护 Markdown 索引管理，但随着文章积累，散落的转发消息缺乏系统性整理、关联和发现能力。

---

## 核心痛点

1. 文章以聊天转发形式存在，无结构化索引
2. wecom-cli 拉取消息只返回标题，原始 URL 丢失
3. 相同主题文章之间没有关联，无法发现遗漏
4. 无法监听感兴趣的公众号，新文章需要手动添加
5. 没有团队分享渠道

---

## 用户需求

### 1. 收藏整理
系统性地收藏、整理已阅读的技术文章，建立结构化的知识体系，而非零散的聊天记录转发。

### 2. 关联推荐
阅读并收藏了一批同一主题的文章（如 10 篇 SDD 编程落地），系统能自动关联出相关的其他文章（如另外 5 篇同类文章）。

### 3. 公众号订阅与监听
发现某个公众号质量高后，能订阅/监听该公众号，新文章自动入库。

### 4. 领域自动匹配与推送
公众号新推文章时，自动匹配到已有知识图谱中的领域，用户每天查看新增文章即可，不需要自己分类。

### 5. 知识领域扩散
基于现有知识体系，自动扩散关联的技术领域，推荐用户可能感兴趣但尚未关注方向的文章。

### 6. 团队分享
整理好的知识库能方便地以静态站形式分享给团队。

---

## 已调研的现有方案

| 类型 | 产品/项目 | 结论 |
|------|-----------|------|
| 商业产品 | 语鲸、腾讯IMA知识库、FOLO | 只覆盖「采集 + 轻度AI总结」，无知识图谱和关联推荐 |
| 开源项目 | WeWe RSS、We-MP-RSS | 可作为订阅抓取组件，无完整方案 |

结论：开源组件足够组装，没有现成的完整方案，需要自建。

---

## 设计决策

| 维度 | 决策 | 理由 |
|------|------|------|
| 部署形态 | 本地 + 轻量服务 | 零运维成本，数据完全私有 |
| 交互方式 | Web UI + VitePress 静态站 | Web 用于日常操作，VitePress 用于团队分享 |
| 关联推荐范围 | 库内 + 订阅源内 | 质量有保障，全网搜索留第二期 |

---

## 非目标（当前版本）

- 全网文章推荐（噪音多，成本高，留第二期）
- 移动端 App
- 企业微信机器人集成
- 多用户支持
