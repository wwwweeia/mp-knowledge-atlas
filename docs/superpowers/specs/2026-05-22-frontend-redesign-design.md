# 前端重设计 Spec

> 日期：2026-05-22
> 状态：待实现

## 背景

当前前端基于 VitePress 默认主题 + D3.js 网络图 iframe，存在三个核心问题：
1. **视觉朴素**：默认文档站样式，无设计感
2. **信息架构混乱**：首页堆砌网络图、桥梁领域、领域表格、文章列表，层次不清
3. **交互体验弱**：纯静态页面，无搜索、筛选、动态效果

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 视觉风格 | Linear 暗色科技 | 深色背景 + 紫色高亮 + 精致边框，科技感强，分享有视觉冲击力 |
| 使用场景 | 实用为主 + 偶尔分享 | 可用性优先，视觉吸引力其次 |
| 首页布局 | 仪表盘式 | 统计卡片 + 缩略网络图 + 领域列表 + 最新文章 |
| 网络图定位 | 嵌入缩略 + 全屏展开 | 日常不挡路，深度探索时全屏 |
| 技术方案 | 轻量 SPA（方案 A） | publish.py 生成 data.json，Vue 3 CDN SPA 消费 |
| SPA 框架 | Vue 3 CDN 引入 | 无构建步骤，与 pipeline 解耦 |

## 架构

### 文件结构

```
site/
├── index.html              # SPA 入口（唯一 HTML）
├── assets/
│   ├── app.js              # Vue 3 SPA 主逻辑（路由 + 组件）
│   ├── app.css             # 全局样式（Linear 暗色主题）
│   └── network-graph.js    # D3.js 网络图模块
├── data/
│   └── data.json           # publish.py 生成的全部数据
└── favicon.ico
```

### 路由

| 路由 | 视图 | 内容 |
|------|------|------|
| `/` | 仪表盘首页 | 统计卡片 + 缩略网络图 + 领域列表 + 最新文章 |
| `/domain/:id` | 领域详情 | 领域信息 + 关键词 + 文章列表（可排序） + 关联领域 |
| `/article/:id` | 文章详情 | 元数据 + 摘要 + 阅读原文按钮 + 同领域推荐 |
| `/graph` | 知识图谱全屏 | D3.js 力导向图（全屏交互） |

## 页面设计

### 全局组件

**导航栏**（固定顶部）：
- 左侧：品牌标识 `◎ 知识地图`
- 右侧：搜索框（`⌘` 快捷键提示）
- 暗色背景 `#111213`，底部 1px 边框 `#1f2023`

### 首页仪表盘（`/`）

四层信息架构：

1. **统计行**：4 个等宽卡片
   - 总文章数 / 领域聚类数 / 桥梁领域数（紫色） / 数据源数
   - 背景 `#131416`，边框 `#1f2023`，大号数字 `#e8e8ed`

2. **双栏主区**：
   - 左侧（50%）：缩略网络图，右下角"全屏展开 →"按钮（紫色胶囊）
   - 右侧（50%）：领域列表，紧凑卡片形式
     - 每行：领域名 + 描述（1行截断） + 文章数（右对齐紫色数字）
     - 桥梁领域：紫色边框 + "桥梁"标签

3. **最新文章**：
   - 紧凑行列表，每行：日期 + 标题 + 所属领域标签（紫色胶囊）
   - 显示最近 10 篇

### 领域详情页（`/domain/:id`）

**Header 区**：
- 桥梁标记（紫色标签） + betweenness 值
- 领域名（22px 粗体）
- 描述文字
- 右上角文章总数（大号紫色数字）
- 关键词胶囊标签（灰色背景、圆角 12px）

**左栏（2/3）— 文章列表**：
- 顶部排序切换：按日期 / 按来源
- 每张文章卡片：标题 + 摘要（2 行截断） + 来源/日期 + ↗ 外链图标
- 点击跳转 `/article/:id`

**右栏（1/3）— 关联领域**：
- 每张关联卡片：领域名 + 文章数 + 相似度百分比 + 紫色进度条
- 按相似度降序排列，显示 top 5

### 文章详情页（`/article/:id`）

- 面包屑：首页 › 领域名 › 文章详情
- 大标题（24px 粗体）
- 元信息行：来源 · 日期 · 所属领域（紫色链接）
- 关键词胶囊标签
- 摘要卡片（独立深色背景区块）
- 紫色 CTA 按钮"阅读原文 ↗"（跳转原始 URL）
- 底部：同领域推荐文章（2-3 篇）

### 全屏知识图谱（`/graph`）

- 全屏 D3.js 力导向图，深色背景
- 左上角：关闭按钮 + "知识图谱"标题（半透明面板 + 毛玻璃效果）
- 右上角：图例面板（桥梁领域/普通领域/关联强度说明）
- 节点：桥梁节点紫色高亮 + 描边，大小区分文章数
- 边：宽度区分关联强度
- 交互：拖拽节点、滚轮缩放、hover tooltip、点击跳转领域详情
- 底部：统计栏 + 操作提示

## 设计语言

### 圆角

- 卡片：8px
- 小元素（按钮、标签）：6px
- 胶囊标签：12px

### 字号

- 页面标题：22-24px
- 卡片标题：13-14px
- 正文：13px
- 辅助文字：11-12px
- 统计数字：28-32px

## 交互行为

- **搜索**：顶栏搜索框，输入即过滤（模糊匹配标题/关键词/领域名），下拉显示匹配结果（文章/领域分组）
- **导航**：所有页面固定顶部导航栏，面包屑回溯
- **页面过渡**：Vue `<transition>` 淡入（200ms）
- **网络图**：拖拽节点、滚轮缩放、hover tooltip、点击跳转
- **响应式**：主要面向桌面端（1280px+），移动端基本适配（单列堆叠）

## publish.py 改造

### 新增

- `generate_data_json()` 函数：从 SQLite（文章/领域）+ JSON（聚类/网络）聚合数据，输出 `site/data/data.json`
- `render_site()` 改为：生成 data.json + 复制 site/ 静态文件

### 删除

- `templates/` 目录（Jinja2 模板）
- VitePress 配置（`site/.vitepress/`、`site/package.json`）
- 生成的 markdown 页面（`site/docs/articles/*.md`、`site/docs/domains/*.md`）

### 不变

- `make refresh` 调用链（pipeline 其他阶段不变）
- `data/` 目录（articles.db、chroma/）
- `src/lib/` 下所有模块

### Makefile 变更

无 VitePress 后，构建和预览命令简化：

- `make build`：原 `cd site && pnpm install && pnpm build` → 改为直接复制 `site/` 到 `out/site/`（纯静态文件，无需构建）
- `make serve`：原 `cd site && pnpm dev` → 改为 `python -m http.server 5173 --directory site/`（或其他静态文件服务器）
